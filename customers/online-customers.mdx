## Overview

The Online Customers page shows all users currently connected to your network, whether through Hotspot or PPPoE. This provides real-time visibility into network usage.

## Accessing Online Customers

Two ways to access:

1. **From Dashboard**: Click the **View** button on the "Online Customers" stat card
2. **From Sidebar**: Click **Online Customers** in the navigation

## What You See

The Online Customers page displays active sessions:

| Column | Description |
|--------|-------------|
| Account No | Customer identifier |
| Phone | Customer phone number |
| IP Address | Assigned IP address |
| MAC Address | Device MAC (Hotspot) |
| Session Start | When connection began |
| Data Used | Bytes uploaded/downloaded |

### Session Information

Each active session shows:

```
Session ID: Unique identifier for this connection
Customer: Account number and phone
IP: Assigned IP address (e.g., 10.10.0.50)
Start Time: When customer connected
Duration: How long they've been connected
Data: Upload/Download statistics
```

## Hotspot vs PPPoE Sessions

### Hotspot Sessions

| Characteristic | Description |
|----------------|-------------|
| Duration | Limited by voucher validity |
| Data | May be capped by plan |
| Device | Identified by MAC address |
| Location | Connected to specific NAS |

### PPPoE Sessions

| Characteristic | Description |
|----------------|-------------|
| Duration | Usually persistent |
| Data | Based on profile limits |
| Device | Customer router/modem |
| Authentication | Username/password |

## Understanding the Data

### Data Usage

```
Upload (TX): Data sent by customer
Download (RX): Data received by customer
Total: Upload + Download
```

### Session Duration

```
Duration = Current Time - Session Start Time
Displayed as: HH:MM:SS or Days/Hours/Minutes
```

## Troubleshooting

| Issue | Possible Cause | Solution |
|-------|----------------|----------|
| Customer shows online but can't browse | DNS or routing issue | Check firewall and DNS settings |
| Duplicate sessions | Reconnection without proper logout | Clear old sessions |
| Customer not showing as online | RADIUS accounting issue | Check RADIUS connectivity |
| Very long session duration | PPPoE keepalive working correctly | Normal for PPPoE |

## Technical Details

### Session Tracking

Sessions are tracked via RADIUS accounting:

```
Accounting-Start: Session begins
Interim-Update: Periodic usage updates
Accounting-Stop: Session ends
```

### Data Sources

| Mode | Source |
|------|--------|
| Hotspot | Mikrotik Hotspot + RADIUS |
| PPPoE | Mikrotik PPPoE Server + RADIUS |
